# Adela-s-ideas
ROUGH IDEAS about math, statistic and teaching


A design-oriented exploration of how mathematical and statistical thinking can support evidence-informed secondary mathematics teaching.


how to bring ai to classroom...


怎么把所学的math&stat和我的specialisation跟教学结合在一起呢？

data anlysis about NAPLAN, VCE scores?

首先肯定不能影响教学任务，课堂环境，behaviour management，二：不建议给学生让他们操作来看自己的得分或者失分情况（老师最好把这事情全做了）

curriculum全是统一的，作业也是？老师的练习题可以不一样，比如有网上出卷（sac考试）（类似题），成绩录入什么的，直接分析错题，以及相对于薄弱的能力，再针对分析和改进......

ps:肯定不能让学生做额外的工作，可是collect成绩的时候以及每到题的错题率的话，人为统计也很累啊，怎么save time呢？

最好开家长会的时候可以给一个书面的report（家长和不会打代码的人要看得懂），以及给上级汇报（数据可视化）


Adela's mind map(informal version):

├─ 1. 目标 Goal
│   ├─ 提高教学决策效率
│   │   Improve efficiency of teaching decisions
│   ├─ 更精准定位学习薄弱点
│   │   Identify learning gaps precisely
│   └─ 不增加学生负担
│       No extra cognitive load for students
│
├─ 2. 现实约束 Constraints（不可突破）
│   ├─ Curriculum 统一
│   ├─ Assessment 不改变
│   ├─ 学生端零额外操作
│   ├─ 不影响 behaviour management
│   └─ 数据仅教师可见
│
├─ 3. 真实问题 Pain Points
│   ├─ 人工统计耗时
│   ├─ 只能看总分 / 平均分
│   ├─ 错误原因靠直觉判断
│   └─ 难以追踪长期趋势
│
├─ 4. 核心理念 Design Principle
│   ├─ 自动化机械统计
│   ├─ 教师保留专业判断
│   └─ Evidence-informed ≠ Data-driven
│
├─ 5. 数据来源 Data Input
│   ├─ Quiz
│   ├─ Topic Test
│   └─ SAC
│
├─ 6. 分析逻辑 Analysis Logic
│   ├─ 描述统计
│   │   ├─ 每题错误率
│   │   ├─ 能力点正确率
│   │   └─ 分位数（非平均分）
│   │
│   ├─ 错误结构分析
│   │   ├─ 概念性错误
│   │   ├─ 程序性错误
│   │   └─ 理解题意错误
│   │
│   └─ 简单纵向比较
│       ├─ Pre vs Post
│       └─ Topic 间变化
│
├─ 7. 教学决策 Instructional Decisions
│   ├─ 是否需要 reteach
│   ├─ 作业题型比例调整
│   ├─ 小范围 intervention
│   └─ 检查调整是否有效
│
├─ 8. 差异化 Differentiation
│   ├─ 题量相同
│   ├─ 题型比例不同
│   └─ 基于 learning needs
│
├─ 9. 沟通 Communication
│   ├─ 家长
│   │   ├─ 语言化结论
│   │   └─ 少数字少图
│   │
│   └─ 学校 / 上级
│       ├─ 数据可视化
│       └─ 支持教学决策
│
├─ 10. 合规与边界 Governance
│   ├─ 不做学生排名
│   ├─ 不贴标签
│   ├─ 不做因果推断
│   └─ 数据仅教学用途
│
└─ 11. 可持续性 Sustainability
    ├─ 不增加日常工作量
    ├─ 可复用
    └─ 长期可维护
